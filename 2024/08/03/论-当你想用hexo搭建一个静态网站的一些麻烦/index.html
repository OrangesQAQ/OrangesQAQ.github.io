论:当你想用hexo搭建一个静态网站的一些麻烦

发表于2024-08-03|更新于2024-08-04
|阅读量:

##1.首先我们要知道hexo是什么
当我们需要了解hexo时，那你肯定是要搭建一个博客了，
那我们从博客出发，搭建博客都需要一个框架，例如wordpress，他就是一个框架，那hexo也自然是一个搭建博客的必须框架
就像这样
>hexo>>博客>>>你的文章等一些东西

##2.我们可以开始安装了
我们已经大致了解完了hexo，那我们就可以开始安装进行食用，可以进去hexo的官网进行查看hexo，是的官网里有npm下载，我相信很多人会使用这个下载方式，但是呢，现在国内地区几乎都被墙了，就算没有被墙下载的速度也慢的一批，那么我们就可以切换npm的源，but现在主流的淘宝源不知什么原因会报错，我们就可以食用腾讯源(https://mirrors.cloud.tencent.com/npm/)
输入npm config set registry https://mirrors.cloud.tencent.com/npm/ 好耶，这下下载速度达到巅峰了
别忘了还要下载node哟awa

##3.关于食用
是的既然安装完了我们就要食用，该如何食用呢
我们可以查看官网或者看我的博客awa，当然就是你现在正在看的东东，
首先我们要在根目录创建一个文件夹名为hexo（昵称随意），让他拥有最高权限预防后续出现问题termux的可以输入chmod 777 hexo/
然后我们解释一下官网要用的指令
###1.hexo init
这个是在你所在的文件夹生成文件，一定要在hexo文件夹里，或者你可以在根目录执行：hexo init hexo 这样文件就生成在hexo文件夹里了，
###2.hexo clean
这个指令表示的是清理缓存会删除缓存文件和删除public文件夹，这边建议写文章写完之后或者换主题之后执行
###3.hexo new [形式] <名字>
这个是表示为新建一个xx名字的什么形式的页面
例如我输入 hexo new page 1
就创建了一个名为1的页面
如果[形式]不做填写，就是默认的文章页面
###4.hexo generate
这个指令表示的是
生成public的文件，会随着一些文件的改动而变动。
可以简写为 hexo g
这是常用的指令。
###5.hexo s
他的意思是自动在本地部署网站，也就是只有自己可以访问，一般的链接ip都是localhost:4000/，我们可以现在这里完成更改再部署到git上，

##4.结尾
看到这里你已经学会了一些皮毛，但是起码可以弄出一个自己访问的博客，接下来我会教你如何将博客部署到github和cloudflare上。Goodbye
